1. Frontend panels (16 screens)

Every screen an operator, engineer, or expert can interact with.

DashboardLive
Real-time operations overview. The first thing an operator sees.

  • Total Entities card — live count split by assets + spaces.
  • Active Findings card — total open findings with critical count highlighted.
  • Change Log Events card — total mutation count with tamper-evident badge.

  • System Health grid — Observations, Capabilities, Mobile Assets, Change Log counts.
  • Feed Controls — start/stop telemetry with live stats (samples processed, findings emitted, latest value).

  • Domain detection — shows classified domain + confidence score.
  • Bundle loading — shows which capability bundle was loaded.
  • Validation result — passed/failed with SHACL violations if any.

2. AI agent system (20 agents, 4 teams)

Every agent in the platform, what it does, and how they connect.

Team 1 — Twin-Building (8 agents)

Takes a user from a sentence to a live twin. The Concierge is the only one that speaks; the rest run silently.

AgentTypeWhat it does
ConciergeLLM, user-facingJargon-free guide. Holds conversation, asks the right next question, decides when enough is known to proceed. The only agent the operator sees.
Vision AgentMultimodalReads uploaded images — floor plans, equipment photos, nameplates — and extracts structured findings (asset counts, types, locations). Uses GPT-4o vision.
Domain ClassifierLLMPicks the vertical (HVAC, maritime, refrigeration, etc.) with a confidence score. Below 0.7 threshold routes back to Concierge for clarification.
Capability ComposerHybridLoads the matching capability bundle from the registry. The plug-and-play moment — makes bundle loading visible.
Schema MapperLLMMaps free-text descriptions + vision findings to concrete ontology entities with canonical types. Falls back to bundle templates without LLM.
ValidatorDeterministicSHACL validation gate. Runs every draft entity through the ontology shapes. Produces human-readable errors for the Concierge to voice.
Graph WriterDeterministicThe only thing that mutates Neo4j. Atomic commit with idempotent UUIDv7 keys. Emits Change Log events.
Scene GeneratorStubExtracts graph topology for 3D rendering. Currently returns entity list; awaiting Three.js integration.

Team 2 — Runtime / Operational (3 agents)

Wakes when findings accumulate into an Incident. No human drives them; operators read their output on the dashboard.

AgentTypeWhat it does
Context GathererDeterministicFetches finding details, downstream dependents, and past incidents from the graph. Populates the context for reasoning.
Diagnosis AgentLLMProduces ranked root-cause hypotheses with confidence scores and evidence. 24-hour cache by incident signature. Writes Diagnosis nodes to the graph.
Recommender AgentLLMProduces ranked recommended actions tagged auto (advisory) or manual (needs human). Advisory-only lock — can never actuate, only recommend.

Team 3 — Bundle Authoring (9 agents)

The verticalization factory. A domain expert creates an entire new capability bundle by conversation — no code.

AgentTypeWhat it does
Industry InterviewerLLM, user-facingConducts an adaptive interview. Builds entity, fault, and measurement catalogues. Never uses jargon.
Ontology DrafterLLMProduces RDF/Turtle fragments extending the platform ontology. Freeze rule: adds downward, never edits core.
Behavior ModelerLLMClassifies each fault as Tier A (physics skeleton), Tier B (baseline config), or Tier C (threshold rule).
Rule AuthorLLMGenerates starter rule library. At least one Tier-C rule so the bundle can fire findings in the demo.
Elicitation DesignerLLMGenerates the question set the Concierge will use for this domain. Team 3's output reconfigures Team 1.
Asset CuratorHybridMatches entities to a 3D asset library. Reports matches and gaps.
LinterDeterministicChecks Turtle parses, freeze rule holds, at least one Tier-C rule exists. Lint failure loops back to Interviewer.
Human Approval GateGateMandatory human approval before publish. Non-negotiable regardless of agent capability.
PublisherDeterministicWrites the signed bundle to the registry via the Plugin SDK. Idempotent on bundle_id.

Team 4 — Platform Extension (5 agents across 2 flows)

Helps platform engineers extend the platform itself.

AgentTypeWhat it does
Plugin InterviewerLLM, user-facingInterviews about which extension point and what the plugin should do.
Plugin ScaffolderHybridGenerates SDK boilerplate for 6 extension points: adapter, behavior, view, webhook, transform, auth.
Pack InterviewerLLM, user-facingInterviews about the target domain and compliance needs for an Accelerator Pack.
Bundle SelectorHybridSelects relevant bundles from the registry and identifies adapter/compliance needs.
Pack AssemblerDeterministicCombines bundles, adapters, and compliance docs into a tender-ready SAP manifest.

The closed loop — the platform's signature feature

1. Bundle Author publishes a new vertical (with a Tier-C rule) → 2. Concierge → Classifier → Composer loads that bundle → 3. Validator → Graph Writer commits a live twin → 4. Feed runs, the Tier-C rule fires a Finding → 5. Diagnosis Agent explains why → Alert on dashboard.

Step 5 pays off step 1. Every component visibly connects.

4. Behavior engine (17 rules, 3 tiers)

The anomaly detection and reasoning system. Behaviors watch telemetry signals, evaluate against models, and emit Findings.

Three tiers working together

TierTypeHow it worksWhat it catches
A — PhysicsFirst-principles equationsPredicts expected values from energy balance / thermal models. Fires when observed diverges from predicted.Faults that violate physical laws — catches what rules and statistics can't see.
B — StatisticalLearned baselinesLearns what "normal" looks like from a warmup window, then flags deviations (z-score). Fires once per anomaly episode.Drift, degradation, load changes — anything that shifts from the learned baseline.
C — RulesHand-written thresholdsFires when a value exceeds a setpoint + offset for a sustained duration. Stateful per entity with debouncing.Obvious threshold violations. The fastest, most deterministic layer.

HVAC behaviors (3)

BehaviorTierWatchesWhat it does
hvac.thermal_physicsAhvac:AirTemperatureLumped-parameter thermal model. T(t) = T_sp + (T_0 - T_sp) × e-t/τ. Fires when residual > 4°C.
hvac.temp_zscoreBhvac:AirTemperatureLearns mean+std from 12 samples, freezes baseline. Fires when z-score > 3σ.
hvac.temp_thresholdChvac:AirTemperatureFires when air temp > setpoint + 3°C sustained for ≥3 minutes.

Diagnosis reasoning chain

Full chain (persisted to graph via single write path)

Finding —[groupedInto]→ Incident —[diagnosedAs]→ Diagnosis —[recommends]→ Recommendation —[proposesAction]→ Action
StepWhat happens
1. CorrelationFindings on the same entity are grouped into an Incident. Severity: critical if any finding is critical.
2. Diagnosis (deterministic)Pattern analysis across behavior IDs and tiers. Confidence 0.6–0.98 based on tier coverage.
3. Diagnosis (LLM-enhanced)Ranked root-cause hypotheses with evidence. Cached 24h by signature. Augments, doesn't replace, the deterministic path.
4. RecommendationRanked actions tagged auto/manual. Advisory-only lock prevents actuation.
5. ActionConcrete step with execution mode (manual/automatic) and status tracking.

5. Infrastructure

Neo4j 5 (Graph Database)
  • Stores every entity, relationship, and finding.
  • Tenant-scoped queries. APOC plugin enabled.
  • Health-checked via Docker. Graceful degradation when offline.
Redis 7 (Event Bus)
  • Redis Streams for per-tenant real-time event fan-out.
  • Best-effort: never blocks the write path. In-memory fallback when Redis is down.
  • SSE endpoint for frontend live subscriptions.
SHACL Validation Gate
  • Every mutation validated against ontology shapes before commit.
  • Inference OFF by design (prevents silent illegal relationship legalization).
  • Returns 422 with human-readable violations.
  • 7/7 gate tests passing.
Graph Writer (Single Write Path)
  • The only code that mutates Neo4j. validate → commit → changelog → event bus.
  • Idempotent on UUIDv7 keys (safe under retries).
  • Methods: create, update, delete, relate.
Change Log (SQLite, Hash-Chained)
  • SHA-256 hash chain per tenant. Tamper-evident.
  • Every mutation recorded: event_id (ULID), action, actor, field_changes.
  • Survives entity deletion. Zero external dependencies (stdlib only).
LLM Gateway
  • Single choke-point for all model calls. Per-session call cap (100).
  • Text completion, JSON completion, vision completion (multimodal).
  • Mandatory deterministic stubs: system works without any API key.
  • Currently OpenAI (gpt-4o-mini / gpt-4o for vision). One-file swap to Claude.
Agent Engine (LangGraph-compatible)
  • In-house StateGraph engine matching the LangGraph API surface.
  • SQLite checkpointer for resumable sessions across process restarts.
  • INTERRUPT_KEY for human-in-the-loop pauses.
  • Designed for one-import-line swap to real LangGraph.
Schema Service
  • Queries the ontology: legal types, predicates, class info, properties, governance.
  • 9 HTTP endpoints under /api/v1/schema/.
  • Feeds the Add Asset modal, the Schema Mapper, and the Copilot (planned).

7. The five disciplines

The engineering principles that hold the platform together across all 20 agents and 47 endpoints.

DisciplineWhat it means
Single write pathOnly the Graph Writer mutates Neo4j. validate → commit → changelog → bus. No agent, no endpoint bypasses it.
State, not callsAgents never call each other. They read/write a shared state object. The graph owns control flow. You can add/remove/reorder agents without touching other agents' code.
Tenant isolation everywhereEvery entity, finding, event, and bus stream is scoped to a tenant. Multi-twin on a single platform instance.
Idempotent commitsUUIDv7 keys + upserts. Safe under checkpoint/Temporal retries. The Graph Writer can be retried without double-writing.
Human gate before publishBundle Author cannot sign without approved=true. Recommender Agent has an advisory-only lock. Safety-critical decisions stay with humans.
