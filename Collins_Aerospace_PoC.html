Section 03
The Six Acts
A narrated demo walkthrough — each act showcases one or more products and feeds into the next.
1
Genesis

Build the Twin

Agentic AI Digital Twin

Collins wants to onboard their Singapore MRO (Maintenance, Repair, Overhaul) facility. The Concierge agent conducts a natural-language interview with the facilities manager.

Behind the scenes:
  • Domain Classifier identifies aerospace-mro with 0.94 confidence
  • Capability Composer loads the aerospace domain pack (vibration monitoring, avionics-grade thermal limits, UPS SoC thresholds)
  • Schema Mapper creates ~15 entities: Building, 2 Floors, 4 Avionics Test Bays, 2 AirHandlers, 1 Chiller, 1 UPS, 1 Transformer, 2 Pumps, 1 Generator, environmental sensors
  • Every entity is SHACL-validated against the ontology before commit
  • GraphWriter writes to Neo4j with full hash-chained changelog
  • Scene Generator creates a navigable 3D GLB model via treemap layout
Demo Moment
The 3D BIM viewer lights up with the full facility. Every entity is clickable — shows canonical type, relationships, and behavior profile. Tenant: collins-sg-mro.
2
Alive

The Facility Comes Alive

Simulation Engine Digital Twin

The dynamics engine starts ticking on every entity in the twin. 15 physics models run simultaneously, topologically sorted so power feeds before it is consumed, cooling feeds before zones reference it.

Physics models running:
  • ZoneThermalModel — ASHRAE RC lumped-capacitance for each test bay, with diurnal outdoor temperature and occupancy profiles
  • ChillerModel — Carnot COP with part-load factor and condition degradation
  • AirHandlerModel — Fan affinity laws (Q∝N, P∝N³), coil first-order lag, filter dust accumulation
  • UPSModel — Peukert battery discharge curves, SoC tracking, cycle counting
  • TransformerModel — IEEE C57.91 oil thermal rise, copper losses

Real-time telemetry flows through the event bus (Redis Streams → SSE → browser). Every graph mutation is visible in the LiveOps panel as it happens.
Demo Moment
Dashboard shows 4 KPIs all green. Sensor cards show live values — Chiller COP: 4.2, UPS SoC: 98%, Supply Air: 21.8°C, Filter ΔP: 120 Pa. Everything nominal. The twin is alive.
3
Fault

The Incident

Simulation Engine

The chiller feeding Avionics Test Bay 2 starts degrading. The simulation engine models condenser fouling — COP drops gradually, supply air temperature rises. Three tiers of detection fire in sequence:

Tier Rule What Fires When
Tier C hvac.temp_threshold Supply air 3°C above setpoint, sustained 3+ minutes T+3 min
Tier B hvac.temp_zscore Temperature z-score exceeds 3σ — statistical anomaly confirmed T+5 min
Tier A hvac.thermal_physics First-order thermal model residual exceeds 4°C — physics divergence T+7 min

Each Finding is written through GraphWriter → changelog → bus. All three findings are auto-grouped into an Incident. The dashboard risk score jumps from NOMINAL to ELEVATED.

Demo Moment
Dashboard turns amber/red. LiveOps shows findings streaming in real-time. The 3D model highlights the chiller in red. Click it — see all three findings with severity pills, timestamps, and the behavior rules that fired.
4
Diagnose

Diagnosis & Remediation

Agentic AI Digital Twin

The operational diagnosis flow activates. Three agents work in sequence to build a full remediation chain.

Agent 1 — Context Gatherer: Pulls all 3 findings on the chiller. Runs BFS traversal (depth=3) to find all dependent equipment: AHU-01 → Test Bay 2 → all sensors in that bay. Checks past incidents on this entity type.

Agent 2 — Diagnosis: Multi-tier corroboration analysis. Threshold + statistical + physics all confirming = confidence 0.92. Root cause: "Chiller COP degradation — condenser fouling or refrigerant charge loss." Creates a Diagnosis node linked to the Incident.

Agent 3 — Recommender: Generates step-by-step remediation:
  • Action 1: Reduce Test Bay 2 load by 40% (immediate, prevent avionics damage)
  • Action 2: Inspect condenser coils within 4 hours
  • Action 3: If fouled — clean coils and recharge refrigerant
  • Action 4: Schedule preventive maintenance cycle every 90 days
Demo Moment
Click through the full chain in the graph: Finding → Incident → Diagnosis → Recommendation → Action. Everything is hash-chained in the changelog. Tamper-evident audit trail.
5
Expand

Autonomous Expansion

Agentic AI Digital Twin Simulation Engine

The diagnosis reveals a critical infrastructure gap: no redundant cooling path. If this chiller fails completely, Test Bay 2 loses environmental control and all in-progress avionics tests are scrapped.

The system acts autonomously:
  • Twin Expand agent adds a backup Chiller entity (chiller-backup-02)
  • Wires it with nxr:feedsInto relationships to the existing cooling manifold
  • Adds a switchover Valve for manual/automatic failover
  • All new entities are SHACL-validated and committed through the same write path
  • The 3D model regenerates to show the backup chiller in the plant room
  • The dynamics engine reloads its topology — picks up new entities automatically
  • Backup chiller starts simulating in "standby" mode immediately
Demo Moment
"We just added redundancy to a live facility without downtime. The simulation is already running on it." The backup chiller sensor card appears — COP nominal, status: standby, ready to take over.

Intel

Digital Twin

Section 06
Agents Activated in This PoC
12 of the 20 agents are directly involved in the Collins demo flow.

Concierge Agent

Natural-language interview with the facilities manager. Elicits facility type, systems, floor count, and equipment. Drives the bundle's question set. Loops on low-confidence classification.

Domain Classifier

Identifies facility domain (aerospace-mro) with confidence scoring. Threshold: 0.7. Falls below → loops back to Concierge for more questions.

Capability Composer

Queries the Bundle Registry for matching domain packs. Loads the aerospace pack's entity templates, behavior rules, and monitoring configurations.

Schema Mapper

Maps conversation findings to ontology entity list. Resolves canonical types against the SHACL-backed ontology. Outputs a validated entity manifest.

Validator Agent

Dry-run SHACL validation on every proposed entity before commit. Catches ontology violations (missing required properties, illegal relationships) before they reach the graph.

Graph Writer Agent

Topologically orders entities (targets before sources), resolves template keys to real IDs, commits through GraphWriter. Registers the twin in TwinRegistry.

Scene Generator

Queries graph for all entities, generates spatial layout via treemap algorithm, creates navigable GLB model with color-coded meshes per entity type.

Context Gatherer

Reads findings, runs BFS traversal (depth=3) for dependent equipment, pulls past incidents on the entity type. Builds the full context packet for diagnosis.

Diagnosis Agent

Multi-tier corroboration analysis. Pattern-matches behavior IDs (threshold + z-score + physics = degradation signature). 24h in-memory cache. Creates Diagnosis node.

Recommender Agent

Generates step-by-step remediation. Creates Recommendation + Action nodes linked to the Diagnosis. Each action has execution mode, priority, and estimated duration.

Twin Expand Agent

Adds new entities to an existing twin without rebuilding. SHACL-validates, commits through GraphWriter, triggers topology reload in the dynamics engine.

Bundle Author (Pre-demo)

Authors the aerospace domain pack before the demo. 9-stage pipeline: interviewer → ontology drafter → rule author → behavior modeler → linter → publisher. Published pack used by the Twin Builder.
